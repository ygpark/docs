## 3. 설정 파일 및 옵션
### 3.1 설정 방법 3가지
- CLI 옵션으로 설정
- package.json의 prettier 필드
- 별도 설정 파일 (.prettierrc)

### 3.2 주요 설정 옵션
#### 3.2.1 기본 포맷팅 옵션
- `printWidth`: 한 줄 최대 글자 수 (기본값: 80)
- `tabWidth`: 탭 너비 (기본값: 2)
- `useTabs`: 탭 vs 공백 사용 여부
- `semi`: 세미콜론 사용 여부
- `singleQuote`: 홑따옴표 vs 쌍따옴표
- `trailingComma`: 후행 쉼표 설정 (none, es5, all)

#### 3.2.2 고급 포맷팅 옵션
- `arrowParens`: 화살표 함수 매개변수 괄호 (always, avoid)
- `bracketSpacing`: 객체 리터럴 괄호 공백
- `endOfLine`: 줄바꿈 방식 (lf, crlf, cr, auto)
- `quoteProps`: 객체 속성 따옴표 (as-needed, consistent, preserve)

#### 3.2.3 언어별 특화 옵션
- `jsxSingleQuote`: JSX에서 홑따옴표 사용
- `jsxBracketSameLine`: JSX 태그 괄호 위치
- `htmlWhitespaceSensitivity`: HTML 공백 처리
- `vueIndentScriptAndStyle`: Vue 파일 스크립트/스타일 들여쓰기

### 3.3 설정 파일 형식
- `.prettierrc` (JSON)
- `.prettierrc.json`
- `.prettierrc.js` (JavaScript)
- `.prettierrc.yml` (YAML)
- `prettier.config.js`

## 4. CLI 사용법
### 4.1 기본 명령어
- 파일 포맷팅: `prettier filename.js`
- 여러 파일 처리: `prettier "src/**/*.js"`
- 제자리 수정: `prettier --write`

### 4.2 주요 CLI 옵션
- `--check`: 포맷팅 확인만 (수정 안함)
- `--list-different`: 포맷팅이 필요한 파일 목록
- `--write`: 파일을 직접 수정
- `--ignore-path`: 무시할 파일 경로 지정
- `--config`: 설정 파일 경로 지정

### 4.3 Glob 패턴 활용
- 파일 패턴 매칭
- 따옴표 사용 주의사항
- 크로스 플랫폼 호환성
